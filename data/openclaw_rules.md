# 赛道一·课题一 主动服务相关要求（比赛规则）
## 一、主动服务核心定义
基于飞书智能助手生态，AI 助手需具备**主动感知需求、主动提供服务、主动执行操作**的能力，而非仅被动响应提问，实现“知识问答 + 指令执行”一体化的主动式服务体验。

## 二、主动服务具体要求
1. **需求主动识别**
   - 能够从用户自然语言指令中主动识别音频后期知识咨询、REAPER 操作指令两类核心需求
   - 对模糊指令主动澄清，对完整指令直接执行，无需用户重复确认

2. **知识主动应答**
   - 针对音频后期相关问题，主动提供专业、准确、结构化的解答
   - 主动补充相关实操技巧、注意事项，提升用户使用效率

3. **操作主动执行**
   - 接收到 REAPER 控制指令后，主动调用对应 Action ID 执行操作
   - 操作完成后主动反馈执行结果，形成交互闭环

4. **场景主动联动**
   - 实现“知识解答 + 软件操作”的主动联动，例如讲解混音技巧后可主动执行对应效果器添加操作
   - 基于音频创作流程，主动提供流程化、步骤式的服务引导

5. **交互主动友好**
   - 遵循飞书交互规范，响应简洁、清晰、无冗余
   - 对异常操作、无效指令主动提示并给出修正建议

6. **服务主动闭环**
   - 确保每一次用户请求都有明确响应与结果
   - 远程控制 REAPER 时主动同步状态，保证操作可追溯、可验证
